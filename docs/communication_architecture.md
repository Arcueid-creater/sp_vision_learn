# 通信架构对比

## 📊 CAN 通信架构

```
┌─────────────────────────────────────────────────────────────┐
│                      主程序层                                │
│  (standard.cpp, sentry.cpp, uav.cpp, etc.)                  │
└────────────────────┬────────────────────────────────────────┘
                     │ 使用
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                   io::CBoard 类                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  公共接口:                                            │  │
│  │  - double bullet_speed                               │  │
│  │  - Mode mode                                         │  │
│  │  - ShootMode shoot_mode                              │  │
│  │  - double ft_angle                                   │  │
│  │  - Eigen::Quaterniond imu_at(timestamp)             │  │
│  │  - void send(Command)                                │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
│  内部实现:                                                   │
│  - SocketCAN can_                                           │
│  - ThreadSafeQueue<IMUData> queue_                          │
│  - void callback(const can_frame & frame)                   │
└────────────────────┬────────────────────────────────────────┘
                     │ 依赖
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                   io::SocketCAN 类                           │
│  - 封装 Linux SocketCAN 接口                                │
│  - 处理 CAN 帧的发送和接收                                   │
│  - 回调机制: callback(can_frame)                            │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                  Linux SocketCAN 驱动                        │
│  - 内核级 CAN 总线支持                                       │
│  - 设备: can0, can1, etc.                                   │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
              [ CAN 总线硬件 ]
                     │
                     ↓
              [ 下位机 STM32 ]
```

### CAN 通信特点

**优势：**
- ✅ 实时性好（硬件级支持）
- ✅ 抗干扰能力强
- ✅ 多节点通信
- ✅ 差分信号，传输距离远

**劣势：**
- ❌ 需要 CAN 收发器硬件
- ❌ 配置复杂（需要设置 CAN 接口）
- ❌ 数据帧长度限制（8 字节）
- ❌ 需要多个 CAN ID 传输完整数据

```
┌─────────────────────────────────────────────────────────────┐
│                      主程序层                                │
│  (standard.cpp, sentry.cpp, uav.cpp, etc.)                  │
└────────────────────┬────────────────────────────────────────┘
                     │ 使用
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                 io::CBoardSerial 类                          │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  公共接口: (与 CBoard 完全相同)                       │  │
│  │  - double bullet_speed                               │  │
│  │  - Mode mode                                         │  │
│  │  - ShootMode shoot_mode                              │  │
│  │  - double ft_angle                                   │  │
│  │  - Eigen::Quaterniond imu_at(timestamp)             │  │
│  │  - void send(Command)                                │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
│  内部实现:                                                   │
│  - std::unique_ptr<VirtualSerial> serial_                   │
│  - ThreadSafeQueue<IMUData> queue_                          │
│  - void callback(const uint8_t * data, uint16_t length)    │
└────────────────────┬────────────────────────────────────────┘
                     │ 依赖
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                 io::VirtualSerial 类                         │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  协议处理:                                            │  │
│  │  - 帧格式: [0xFF][0xAA][长度][数据][CRC16]          │  │
│  │  - CRC16-CCITT 校验                                  │  │
│  │  - 状态机解析                                         │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
│  线程管理:                                                   │
│  - read_thread_: 持续读取串口数据                           │
│  - daemon_thread_: 监控连接状态，自动重连                   │
│                                                              │
│  回调机制:                                                   │
│  - std::function<void(const uint8_t*, uint16_t)> rx_handler_│
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                  Linux 串口驱动 (termios)                    │
│  - 用户空间串口接口                                          │
│  - 设备: /dev/ttyACM0, /dev/ttyUSB0, etc.                  │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
              [ USB 虚拟串口 ]
                     │
                     ↓
              [ 下位机 STM32 ]
```

### 串口通信特点

**优势：**
- ✅ 硬件简单（USB 即可）
- ✅ 配置简单（只需设备路径和波特率）
- ✅ 批量传输（单帧可传输大量数据）
- ✅ CRC16 校验，可靠性高
- ✅ 自动重连机制
- ✅ 状态机解析，抗干扰

**劣势：**
- ❌ 实时性略低于 CAN（用户空间轮询）
- ❌ 点对点通信（不支持多节点）
- ❌ 传输距离受限（USB 线缆长度）

## 🔄 接口兼容性

### 完全兼容的公共接口

```cpp
// CBoard 和 CBoardSerial 提供相同的接口
class CBoard / CBoardSerial {
public:
  // 状态变量
  double bullet_speed;      // 弹速 (m/s)
  Mode mode;                // 工作模式
  ShootMode shoot_mode;     // 射击模式（哨兵）
  double ft_angle;          // FT角度（无人机）
  
  // 方法
  Eigen::Quaterniond imu_at(timestamp);  // 获取IMU数据（带插值）
  void send(Command command);             // 发送控制命令
};
```

### 使用示例（完全相同）

```cpp
// 主程序中的使用方式完全一致
int main() {
  // CAN 方式
  io::CBoard cboard(config_path);
  
  // 串口方式（只需修改类名）
  io::CBoardSerial cboard(config_path);
  
  // 使用方式完全相同
  while (true) {
    auto q = cboard.imu_at(timestamp);
    auto speed = cboard.bullet_speed;
    auto mode = cboard.mode;
    
    // ... 视觉处理 ...
    
    cboard.send(command);
  }
}
```

## 📦 数据格式对比

### CAN 数据格式

**四元数数据（CAN ID: 0x100）**
```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│ X高 │ X低 │ Y高 │ Y低 │ Z高 │ Z低 │ W高 │ W低 │
└────┴────┴────┴────┴────┴────┴────┴────┘
  int16  int16  int16  int16  (× 1e4)
```

**弹速和模式（CAN ID: 0x101）**
```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│速度高│速度低│模式 │射击 │FT高 │FT低 │ -- │ -- │
└────┴────┴────┴────┴────┴────┴────┴────┘
  int16   uint8 uint8 int16
```

**控制命令（CAN ID: 0xFF）**
```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│控制 │射击 │Yaw高│Yaw低│Pit高│Pit低│Dis高│Dis低│
└────┴────┴────┴────┴────┴────┴────┴────┘
  uint8 uint8 int16  int16  int16
```

### 串口数据格式

**帧结构**
```
┌────┬────┬────┬────┬──────────┬────┬────┐
│0xFF│0xAA│长度L│长度H│  数据(N)  │CRC低│CRC高│
└────┴────┴────┴────┴──────────┴────┴────┘
  固定帧头   小端序    N字节      CRC16
```

**STM32 状态数据（消息 ID: 0x01）**
```
┌────┬──────┬──────┬──────┬──────┬──────┬────┬────┐
│0x01│ X(4) │ Y(4) │ Z(4) │ W(4) │速度(4)│模式│射击│
└────┴──────┴──────┴──────┴──────┴──────┴────┴────┘
  ID   float  float  float  float  float  u8   u8
       四元数 (16字节)              弹速   模式 射击
```

**视觉控制命令（消息 ID: 0x02）**
```
┌────┬────┬────┬──────┬──────┐
│0x02│控制│射击│Yaw(4)│Pit(4)│
└────┴────┴────┴──────┴──────┘
  ID   u8   u8   float  float
```

## 🔀 迁移影响分析

### 不受影响的部分 ✅

- ✅ 视觉算法（detector, tracker, aimer, solver）
- ✅ 相机接口（io::Camera）
- ✅ 数据结构（Command, Target, Armor）
- ✅ 工具类（logger, plotter, recorder）

## 🎯 推荐迁移策略

### 策略一：完全切换（推荐）

**适用场景：** 确定只使用串口通信

**优势：**
- 代码简洁，无冗余
- 维护成本低
- 配置简单

**步骤：**
1. 使用自动化脚本 `migrate_to_serial.sh`
2. 编译测试
3. 验证功能

### 策略二：保留两种方式

**适用场景：** 需要在不同硬件平台切换

**优势：**
- 灵活性高
- 兼容多种硬件
- 便于调试对比

**步骤：**
1. 使用 `io/cboard_wrapper.hpp`
2. 修改 CMakeLists.txt 添加编译选项
3. 通过宏定义切换

### 策略三：渐进式迁移

**适用场景：** 大型项目，需要逐步验证

**优势：**
- 风险低
- 便于问题定位
- 可随时回滚

**步骤：**
1. 先迁移测试程序
2. 再迁移调试程序
3. 最后迁移正式程序

## 📈 性能对比

| 指标 | CAN 通信 | 串口通信 |
|------|---------|---------|
| 延迟 | ~1ms | ~2-3ms |
| 吞吐量 | 1 Mbps | 115200 bps |
| CPU 占用 | 低（内核处理） | 中（用户空间轮询） |
| 可靠性 | 高（硬件CRC） | 高（CRC16校验） |
| 配置复杂度 | 高 | 低 |
| 硬件成本 | 高（需CAN收发器） | 低（USB即可） |

**结论：** 对于机器人视觉系统，串口通信的性能完全满足需求（相机帧率通常 < 200 Hz）。

```bash
# 查看 CAN 接口
ip link show can0

# 监控 CAN 数据
candump can0

# 发送测试数据
cansend can0 100#0102030405060708
```

```bash
# 查看串口设备

# 监控串口数据
sudo cat /dev/ttyACM0 | hexdump -C

sudo minicom -D /dev/ttyACM0 -b 115200
```
