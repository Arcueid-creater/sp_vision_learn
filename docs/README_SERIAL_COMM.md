## 🔄 数据流向

### 下位机 → 上位机

| 数据类型 | 消息ID | 频率 | 用途 |
|---------|--------|------|------|
| IMU数据 | 0x01 | 100-200Hz | 姿态估计、目标跟踪 |
| 弹速和模式 | 0x02 | 10-50Hz | 弹道计算、模式切换 |
| 遥控器+电机 | 0x10 | 50Hz | 调试、监控 |

### 上位机 → 下位机

| 数据类型 | 消息ID | 频率 | 用途 |
|---------|--------|------|------|
| 云台命令 | 0x03 | 按需 | 云台控制、射击控制 |
| 底盘速度 | 0x20 | 按需 | 底盘运动控制 |

## ⚠️ 注意事项

### 安全性

1. **权限设置**：确保用户有串口访问权限
   ```bash
   sudo usermod -a -G dialout $USER
   ```

2. **CRC校验**：始终启用CRC16校验，确保数据完整性

3. **错误处理**：实现超时检测和重连机制

### 可靠性

1. **断线重连**：上位机自动重连，无需手动干预
2. **队列缓冲**：防止数据丢失
3. **状态机解析**：防止数据错位

### 性能

1. **发送频率**：不要超过推荐频率，避免缓冲区溢出
2. **DMA传输**：下位机使用DMA减少CPU占用
3. **批量发送**：合并小数据包，减少协议开销

## 🐛 故障排查

### 问题1：无法打开串口

**症状**：`Failed to open serial port: /dev/ttyACM0`

**原因**：
- 设备不存在
- 权限不足
- 设备被占用

**解决**：
```bash
# 检查设备是否存在
ls /dev/ttyACM*

# 设置权限
sudo chmod 666 /dev/ttyACM0

# 检查是否被占用
lsof /dev/ttyACM0
```

---

### 问题2：CRC校验失败

**症状**：`CRC16 mismatch! Received: 0x1234, Calculated: 0x5678`

**原因**：
- 数据传输错误
- 上下位机CRC算法不一致
- 字节序不一致

**解决**：
- 检查USB连接
- 降低发送频率
- 确保使用相同的CRC16-CCITT算法
- 确保使用小端序

---

### 问题3：数据丢失

**症状**：部分数据未接收到

**原因**：
- 发送频率过高
- 缓冲区溢出
- USB带宽不足

**解决**：
- 降低发送频率
- 增加缓冲区大小
- 使用DMA传输
- 批量发送数据
