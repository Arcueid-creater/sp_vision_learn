# USB虚拟串口通信协议迁移总结

## 修改概述

本次修改将通信协议从CAN总线迁移到USB虚拟串口，并统一了上下位机的消息格式，确保数据传输的完整性和兼容性。

---

## 一、修改的文件

### 下位机端（STM32）

1. **trans_task.c**
   - 添加了 `send_imu_data()` 函数：发送IMU四元数数据
   - 添加了 `send_bullet_speed()` 函数：发送弹速和模式信息
   - 删除了 `send_single_motor()` 函数（不再需要）
   - 更新了消息ID枚举，添加了：
     - `TRANS_MSG_ID_IMU_DATA = 0x01`
     - `TRANS_MSG_ID_BULLET_SPEED = 0x02`
     - `TRANS_MSG_ID_GIMBAL_COMMAND = 0x03`
   - 在 `process_usb_bytes()` 中添加了云台命令接收处理
   - 保留了底盘速度命令接收处理（已有）
   - 保留了遥控器数据发送（已有）

2. **trans_task.h**
   - 添加了 `send_imu_data()` 函数声明
   - 添加了 `send_bullet_speed()` 函数声明

     - 发送云台控制命令

   - 无需修改（已经实现了完整的协议栈）

### 文档

## 二、协议消息类型

### 消息ID定义

| ID | 名称 | 方向 | 说明 | 数据长度 |
|----|------|------|------|----------|
| 0x01 | IMU_DATA | 下位机→上位机 | IMU四元数 | 17字节 |
| 0x02 | BULLET_SPEED | 下位机→上位机 | 弹速和模式 | 7字节 |
| 0x03 | GIMBAL_COMMAND | 上位机→下位机 | 云台控制命令 | 11字节 |
| 0x10 | RC_DBUS | 下位机→上位机 | 遥控器和电机数据 | 69字节 |
| 0x20 | CHASSIS_CMD | 上位机→下位机 | 底盘速度命令 | 13字节 |

## 四、关键改进

### 4.1 统一的消息格式

所有消息都遵循统一的帧格式：

```
[0xFF][0xAA][长度(2)][消息ID(1)][数据(N)][CRC16(2)]
```

### 4.2 完整的数据类型支持

- **IMU数据**：四元数（x, y, z, w）
- **弹速和模式**：弹速 + 机器人模式 + 射击模式
- **云台命令**：控制标志 + 射击标志 + Yaw + Pitch
- **底盘命令**：线速度x + 线速度y + 角速度z
- **遥控器数据**：完整的遥控器状态 + 4个电机反馈

### 4.3 可靠性保证

- **CRC16校验**：确保数据完整性
- **状态机解析**：防止数据错位
- **断线重连**：自动恢复连接
- **队列缓冲**：防止数据丢失

### 4.4 高性能

- **批量传输**：一次传输多个数据
- **DMA支持**：减少CPU占用
- **线程安全**：支持多线程访问

## 五、使用示例对比

### 5.1 下位机发送数据

#### 原来（CAN）：
```c
// 需要手动分包，每次最多8字节
can_send_data(0x201, data, 8);
```

#### 现在（USB串口）：
```c
// 直接发送完整数据，自动封装
send_imu_data(qx, qy, qz, qw);
send_bullet_speed(speed, mode, shoot_mode);
```

### 5.2 上位机接收数据

#### 原来（CAN）：
```cpp
// 需要手动解析CAN帧
void parse_can_frame(uint32_t id, uint8_t* data, uint8_t len) {
    // 复杂的解析逻辑...
}
```

#### 现在（USB串口）：
```cpp
// 自动解析，直接使用
Eigen::Quaterniond q = cboard.imu_at(timestamp);
double bullet_speed = cboard.bullet_speed;
```

## 六、兼容性说明

### 6.1 向后兼容

- 保留了原有的遥控器数据格式（0x10）
- 保留了底盘命令格式（0x20）
- 其他模块无需修改即可使用

### 6.2 新增功能

- IMU数据传输（0x01）
- 弹速和模式传输（0x02）
- 云台命令传输（0x03）

### 6.3 不兼容的部分

- 物理接口从CAN改为USB
- 需要重新配置设备路径（`/dev/ttyACM0`）
- 需要确保USB CDC驱动正常工作

## 九、常见问题

### Q1：为什么要从CAN迁移到USB串口？

**A**：
- USB传输速率更高（12Mbps vs 1Mbps）
- 单帧数据长度更大（512字节 vs 8字节）
- 无需额外的CAN收发器硬件
- 更容易调试和监控

### Q2：USB串口的可靠性如何？

**A**：
- 使用CRC16校验确保数据完整性
- 状态机解析防止数据错位
- 自动断线重连机制
- 队列缓冲防止数据丢失

### Q3：如何确保上下位机数据格式一致？

**A**：
- 使用统一的消息ID定义
- 使用相同的CRC16算法
- 使用相同的字节序（小端序）
- 参考 `docs/serial_protocol.md` 文档

### Q4：如何调试通信问题？

**A**：
1. 使用串口调试工具查看原始数据
2. 检查设备路径是否正确
3. 检查波特率是否匹配
4. 查看日志输出
5. 使用示波器或逻辑分析仪
